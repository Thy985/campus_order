# 校园点餐管理系统技术选型方案

## 1. 技术选型原则

### 1.1 选型标准
- **成熟稳定**：选择经过生产环境验证的技术栈
- **开发效率**：提升团队开发效率，降低学习成本
- **维护简单**：代码清晰，易于维护
- **成本可控**：考虑开发成本、运维成本和授权成本
- **功能够用**：满足基本业务需求

### 1.2 技术栈概览
```
前端层：Vue3（单端应用）
后端层：Spring Boot 3.3.0
数据层：MySQL 8.0
缓存：内存缓存（可选Redis单机）
部署：单机部署
```

## 2. 后端技术选型

### 2.1 开发语言
**选型：Java 17**

### 2.2 开发框架
**选型：Spring Boot 3.3.0**

**核心组件：**
- **Spring Boot 3.3.0**：快速开发框架
- **Sa-Token**：安全认证
- **MyBatis-Flex**：数据访问
- **Spring AI**：人工智能功能
- **Spring Validation**：参数验证
- **Spring Web**：Web服务

**架构模式：**
```
单体应用架构（Monolith）
- 控制器层（Controller）
- 服务层（Service）
- 数据访问层（DAO）
- 实体层（Entity）
```

### 2.3 数据库选型

#### 2.3.1 关系型数据库
**选型：MySQL 8.0**

**选择理由：**
- 开源免费，社区活跃
- 性能优秀，支持中小型应用
- ACID事务支持
- 简单易用，学习成本低
- 丰富的存储引擎（InnoDB）

**配置方案：**
- **架构**：单库单实例
- **存储引擎**：InnoDB（支持事务、行锁）
- **字符集**：utf8mb4（支持emoji）
- **连接池**：HikariCP（性能最优）

**替代方案：**
- SQLite：轻量级，适合小型应用
- PostgreSQL：功能更强大，但生态不如MySQL

#### 2.3.2 缓存（可选）
**选型：内存缓存（Caffeine）或Redis单机**

**使用场景：**
- **缓存**：热点数据缓存（菜品信息、商家信息）
- **会话**：用户Session存储

**部署方案（Redis）：**
- 单机模式
- 持久化：RDB模式
- 最大内存：2GB

### 2.4 文件存储
**选型：本地文件系统**

**使用场景：**
- 菜品图片存储
- 用户头像

**存储策略：**
- 图片压缩：上传时自动压缩
- 目录结构：按日期分目录存储

## 5. 部署方案

### 5.1 部署方式
**选型：单机部署**

**部署流程：**
```
代码编译 → 打包成jar → 上传服务器 → 启动应用
```

### 5.2 服务器配置

#### 开发环境（单机）
```
配置：2核4G
服务：应用 + MySQL
```

#### 生产环境（单机）
```
配置：4核8G
服务：应用 + MySQL
```

### 5.3 备份策略
- **数据库备份**：每日全量备份
- **文件备份**：每周全量备份

## 6. 安全方案

### 6.1 认证授权
- **Sa-Token**：安全认证，支持Token认证、权限控制
- **RBAC**：基于角色的权限控制

### 6.2 数据安全
- **传输加密**：HTTPS（TLS 1.2）
- **密码加密**：BCrypt（加盐哈希）
- **SQL防注入**：参数化查询、ORM框架

### 6.3 接口安全
- **请求验证**：参数校验
- **防止XSS**：输入过滤

## 7. 性能优化方案

### 7.1 缓存策略
- **本地缓存**：Caffeine缓存热点数据
- **缓存更新**：手动更新或定时刷新

### 7.2 数据库优化
- **索引优化**：添加必要的索引
- **查询优化**：减少复杂查询
- **连接池优化**：合理配置连接池参数

### 7.3 接口优化
- **批量查询**：减少数据库查询次数
- **分页查询**：大数据量分页

## 8. 技术选型总结

### 8.1 技术栈清单
```yaml
后端框架:
  - Spring Boot 4.0.2
  - MyBatis-Flex 1.0.0
  - Sa-Token 1.34.0
  - Spring AI 1.0.0

数据存储:
  - MySQL 8.0
  - 本地文件系统（图片存储）

缓存（可选）:
  - Caffeine（内存缓存）
  - Redis 7.0（单机）

开发工具:
  - IntelliJ IDEA
  - Maven
  - SpringDoc OpenAPI
  - Postman

部署:
  - 单机部署
  - 手动备份
```

### 8.2 技术选型优势
1. **简单易用**：技术栈简洁，学习成本低
2. **开发高效**：Spring Boot快速开发，减少配置工作
3. **维护成本低**：单体应用架构，部署简单
4. **成本可控**：开源方案为主，降低服务器成本
5. **功能够用**：满足基本业务需求

### 8.3 技术风险与应对
| 风险 | 应对措施 |
|------|----------|
| 性能瓶颈 | 合理使用缓存、优化数据库查询 |
| 单点故障 | 定期备份数据、制定恢复方案 |
| 扩展性受限 | 预留接口，便于后续扩展 |

## 9. Spring AI模块设计

### 9.1 核心功能

#### 9.1.1 智能推荐系统
- **功能描述**：基于用户历史订单、浏览记录，推荐个性化菜品
- **技术实现**：Spring AI + 协同过滤算法
- **使用场景**：用户首页推荐、商家详情页推荐

#### 9.1.2 智能客服
- **功能描述**：AI驱动的智能客服，回答用户常见问题
- **技术实现**：Spring AI + 对话模型
- **使用场景**：用户咨询、订单状态查询

#### 9.1.3 菜品识别
- **功能描述**：上传菜品图片，自动识别菜品信息
- **技术实现**：Spring AI + 图像识别模型
- **使用场景**：商家上传菜品图片时自动填写信息

#### 9.1.4 智能菜单生成
- **功能描述**：根据季节、热门趋势，智能生成推荐菜单
- **技术实现**：Spring AI + 生成模型
- **使用场景**：商家菜单管理

### 9.2 技术架构

#### 9.2.1 架构模式
```
AI服务层
├── 推荐服务
├── 客服服务
├── 图像识别服务
└── 生成服务
    ↓
Spring AI
    ↓
AI模型（本地/云服务）
```

#### 9.2.2 模型选择
- **本地模型**：适合离线部署，响应速度快
- **云服务模型**：功能更强大，适合复杂场景

### 9.3 集成方案

#### 9.3.1 API设计
- **推荐API**：`/api/v1/ai/recommend`
- **客服API**：`/api/v1/ai/chat`
- **图像识别API**：`/api/v1/ai/recognize`
- **菜单生成API**：`/api/v1/ai/menu/generate`

#### 9.3.2 性能优化
- **缓存策略**：缓存推荐结果、常见问题回答
- **异步处理**：复杂AI任务异步执行
- **模型优化**：选择轻量级模型，提升响应速度

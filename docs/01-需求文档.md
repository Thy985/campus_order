# 校园点餐管理系统需求文档（降级版）

## 1. 项目概述

### 1.1 项目背景
校园点餐管理系统旨在为高校师生提供便捷的线上点餐服务，解决传统食堂排队时间长、点餐效率低等问题，同时为食堂商家提供基本的订单管理能力。

### 1.2 项目目标
- 提升师生点餐体验，减少排队等待时间
- 优化食堂运营效率，降低人工成本
- 实现基本的线上点餐和订单管理功能
- 支持单一食堂的档口管理

## 2. 用户角色定义

### 2.1 学生/教师（C端用户）
- **主要需求**：快速点餐、在线支付、订单追踪
- **使用场景**：课间点餐、到店取餐、查看历史订单

### 2.2 商家（食堂档口）
- **主要需求**：接单管理、菜品管理、基本营业统计
- **使用场景**：实时接单、修改菜品状态、查看简单销售数据

### 2.3 管理员
- **主要需求**：基础用户管理、订单监控、简单系统配置
- **使用场景**：用户管理、订单查询、基本参数设置

## 4. 核心业务流程

### 4.1 用户点餐流程
```
1. 用户登录 → 2. 选择档口 → 3. 浏览菜品 → 4. 加入购物车
→ 5. 确认订单 → 6. 在线支付 → 7. 等待接单
→ 8. 商家制作 → 9. 到店取餐 → 10. 确认完成
```

### 4.2 商家接单流程
```
1. 接收订单通知 → 2. 查看订单详情 → 3. 确认接单/拒绝
→ 4. 开始制作 → 5. 更新订单状态 → 6. 用户取餐 → 7. 订单完成
```

## 5. 非功能性需求

### 5.1 性能需求
- 系统响应时间：普通请求 < 1s
- 并发支持：支持200+ QPS（午餐高峰期）
- 数据库查询：单表查询 < 200ms
- 页面加载：首屏加载 < 3s

### 5.2 可用性需求
- 系统可用性：99.5%
- 故障恢复时间：< 1小时
- 数据备份：每日全量备份

### 5.3 安全性需求
- 数据传输：HTTPS加密
- 密码存储：BCrypt加密
- 支付安全：接入第三方支付平台，不存储敏感信息
- SQL注入防护：参数化查询

### 5.4 兼容性需求
- 移动端：iOS 12+、Android 8+
- 浏览器：Chrome、Safari、Edge最新版本

## 6. 数据需求

### 6.1 数据量预估
- 用户量：1,000 - 5,000（中小型校园）
- 商家数：5 - 20个档口
- 日订单量：200 - 1,000单
- 菜品数：100 - 500个

### 6.2 数据保留策略
- 订单数据：保留1年
- 系统日志：保留1个月
- 临时数据（购物车）：7天自动清理

## 7. 约束条件

### 7.1 技术约束
- 支持基本并发场景（午餐/晚餐高峰）

### 7.2 业务约束
- 营业时间限制：早餐 6:30-9:00、午餐 10:30-13:30、晚餐 16:30-19:30
- 订单超时自动取消：15分钟未支付

### 7.3 合规约束
- 用户隐私保护（符合《个人信息保护法》）
- 支付合规（接入持牌支付机构）

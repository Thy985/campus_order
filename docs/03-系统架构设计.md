# 校园点餐管理系统架构设计（降级版）

## 1. 架构设计原则

### 1.1 设计原则
- **简单实用**：满足基本业务需求，避免过度设计
- **开发高效**：提升团队开发效率，降低学习成本
- **维护简单**：代码清晰，易于理解和修改
- **成本可控**：考虑开发成本、运维成本和授权成本
- **安全可靠**：数据安全、接口安全、支付安全

### 1.2 架构风格
- **单体应用架构**：所有功能集成在一个应用中
- **分层架构**：控制层、服务层、数据访问层
- **同步调用**：简化业务流程，减少复杂性

### 2.2 架构分层说明

#### 2.2.1 客户端层
- **Web端**：统一的Web应用（学生/教师/商家/管理员使用）
- **访问方式**：浏览器访问

#### 2.2.2 应用层
- **Spring Boot 4.0.2开发框架
- **控制层**：处理HTTP请求，参数验证，响应格式化
- **服务层**：业务逻辑处理
- **数据访问层**：数据库操作，使用MyBatis-Flex
- **实体层**：数据模型定义

#### 2.2.3 数据层
- **MySQL 8.0**：关系型数据库，单库单实例
- **本地文件系统**：存储图片等静态资源

#### 2.2.4 基础设施层
- **单机部署**：应用和数据库部署在同一台服务器
- **简单备份**：定期数据库备份

## 3. 核心模块设计

### 3.1 模块划分

#### 3.1.1 用户模块
**职责：** 用户账号管理、认证授权

**核心功能：**
- 用户注册/登录（手机号）
- 个人信息管理

**数据库表：**
- user（用户表）

**关键类：**
- UserController：处理用户相关请求
- UserService：用户业务逻辑
- UserMapper：用户数据访问

#### 3.1.2 商家模块
**职责：** 商家信息管理、店铺运营

**核心功能：**
- 店铺信息管理
- 营业时间设置
- 商家分类管理

**数据库表：**
- merchant（商家表）
- merchant_category（商家分类表）
- merchant_business_hours（营业时间表）

**关键类：**
- MerchantController：处理商家相关请求
- MerchantService：商家业务逻辑
- MerchantMapper：商家数据访问

#### 3.1.3 商品模块
**职责：** 菜品管理、库存管理

**核心功能：**
- 菜品CRUD
- 菜品分类管理
- 库存管理（扣减、回滚）

**数据库表：**
- product（商品表）
- product_category（商品分类表）

**关键类：**
- ProductController：处理商品相关请求
- ProductService：商品业务逻辑
- ProductMapper：商品数据访问

#### 3.1.4 订单模块
**职责：** 订单全生命周期管理

**核心功能：**
- 订单创建（购物车 → 订单）
- 订单支付
- 订单状态流转
- 订单查询（用户、商家）

**数据库表：**
- order（订单表）
- order_item（订单明细表）

**订单状态机：**
```
待支付 → 待接单 → 制作中 → 待取餐 → 已完成
   ↓        ↓        ↓
已取消   已取消   已取消
```

**关键类：**
- OrderController：处理订单相关请求
- OrderService：订单业务逻辑
- OrderMapper：订单数据访问

#### 3.1.5 支付模块
**职责：** 支付流程管理

**核心功能：**
- 统一支付接口（微信、支付宝）
- 支付回调处理

**数据库表：**
- payment（支付表）

**支付流程：**
```
1. 订单模块创建支付单
2. 调用第三方支付接口
3. 返回支付参数给前端
4. 用户完成支付
5. 第三方回调支付模块
6. 支付模块验证签名
7. 更新支付状态
8. 订单模块更新订单状态
```

**关键类：**
- PaymentController：处理支付相关请求
- PaymentService：支付业务逻辑
- PaymentMapper：支付数据访问

#### 3.1.6 AI模块
**职责：** 智能功能支持

**核心功能：**
- 智能推荐：基于用户历史行为推荐菜品
- 智能客服：回答用户常见问题
- 菜品识别：上传图片自动识别菜品
- 智能菜单生成：根据季节、趋势生成推荐菜单

**关键类：**
- AiController：处理AI相关请求
- AiService：AI业务逻辑
- RecommendationService：推荐服务
- ChatService：客服服务
- ImageRecognitionService：图像识别服务
- MenuGenerationService：菜单生成服务

## 5. 安全架构设计

### 5.1 认证授权
```
┌─────────┐
│ 客户端  │
└────┬────┘
     │ 1. 登录请求
     ↓
┌─────────────┐
│ 认证组件    │
│ (Session)   │
└──────┬──────┘
       │ 2. 生成Session
       ↓
┌─────────────┐
│ 业务逻辑    │
│ (权限校验)   │
└─────────────┘
```

### 5.2 接口安全
- **HTTPS**：全站HTTPS
- **请求验证**：参数校验
- **SQL防注入**：MyBatis参数化查询
- **XSS防护**：输入过滤、输出转义

### 5.3 数据安全
- **密码加密**：BCrypt（加盐哈希）
- **敏感数据**：手机号等敏感信息脱敏处理

## 7. 部署架构设计

### 7.1 单机部署

#### 7.1.1 服务器配置
```
开发环境：2核4G
生产环境：4核8G
```

#### 7.1.2 部署流程
```
1. 编译打包：mvn clean package
2. 上传jar包到服务器
3. 启动应用：java -jar campus-order.jar
4. 配置数据库连接
5. 配置静态文件存储路径
```

### 7.2 环境配置

#### 7.2.1 配置文件
```yaml
# application.yml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/campus_order_db
    username: root
    password: password
  
  # 文件上传配置
  servlet:
    multipart:
      max-file-size: 10MB
      max-request-size: 10MB

# 图片存储路径
file:
  upload-dir: /data/upload
```

## 8. 监控与运维

### 8.1 日志管理
```
日志框架：SLF4J + Logback
日志级别：ERROR/WARN/INFO
日志文件：按日期滚动
存储路径：/data/logs
```

### 8.2 健康检查
- **Spring Boot Actuator**：应用健康检查
- **访问地址**：/actuator/health

### 8.3 常见问题排查
```
1. 数据库连接失败：检查数据库服务状态和连接配置
2. 应用启动失败：查看日志文件中的错误信息
3. 接口响应慢：检查SQL语句，添加索引
4. 图片上传失败：检查文件存储路径权限
```

- 单体应用架构 + 分层设计

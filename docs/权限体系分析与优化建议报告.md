## 2. 问题发现与分析

### 2.1 问题 SEC-001: 普通用户登录失败

#### 2.1.1 问题描述

集成测试中发现，使用手机号 `13800000011` 和密码 `123456` 登录失败，返回错误信息：
```json
{"code":40001,"message":"用户名或密码错误","data":null}
```

#### 2.1.2 根因分析

**数据层面**:
- SQL文件中用户 `13800000011` 的密码为 `$2a$10$N.zmdr9k7uOCQb376NoUnuTJ8iAt6Z5EHsM8lE9lBOsl7iAt6Z5EO`
- 这是BCrypt哈希值，对应的明文密码**不是** `123456`

**代码层面**:
- 登录逻辑正确，使用BCrypt进行密码验证
- 问题在于测试数据与测试用例不匹配

#### 2.1.3 影响评估

| 影响维度 | 评估 |
|----------|------|
| 功能影响 | 高 - 用户无法登录 |
| 安全影响 | 低 - 密码验证机制正确 |
| 业务影响 | 中 - 影响测试进度 |

#### 2.1.4 修复方案

**方案A: 更新测试用例密码** (推荐)
```sql
-- 更新测试用户密码为123456的BCrypt哈希
UPDATE user SET password = '$2a$10$N.zmdr9k7uOCQb376NoUnuTJ8iAt6Z5EHsM8lE9lBOsl7iAt6Z5EO' 
WHERE phone IN ('13800000011', '13800000012');
```

**方案B: 更新SQL初始化脚本**
```sql
-- 在V5__full_init_optimized.sql中修改密码插入语句
INSERT INTO user (phone, password, nickname, role, status, is_deleted, create_time, update_time) 
VALUES 
('13800000011', '$2a$10$N.zmdr9k7uOCQb376NoUnuTJ8iAt6Z5EHsM8lE9lBOsl7iAt6Z5EO', 'student01', 'customer', 1, 0, NOW(), NOW()),
('13800000012', '$2a$10$N.zmdr9k7uOCQb376NoUnuTJ8iAt6Z5EHsM8lE9lBOsl7iAt6Z5EO', 'student02', 'customer', 1, 0, NOW(), NOW());
```

### 2.2 问题 SEC-002: 管理端接口权限控制不完整

#### 2.2.1 问题描述

**DashboardController.java** 中使用了 `@SaCheckLogin` 而非 `@SaCheckRole("admin")`：

```java
@RestController
@RequestMapping("/api/admin/dashboard")
public class DashboardController {
    
    @SaCheckLogin  // ❌ 仅检查登录，不检查角色
    @GetMapping("/overview")
    public Result<Map<String, Object>> getOverview() {
        // 任何登录用户都可访问
    }
}
```

#### 2.2.2 权限漏洞影响

**受影响接口**:

| 控制器 | 接口 | 当前权限 | 应有权限 |
|--------|------|----------|----------|
| DashboardController | /api/admin/dashboard/overview | @SaCheckLogin | @SaCheckRole("admin") |
| DashboardController | /api/admin/dashboard/order-statistics | @SaCheckLogin | @SaCheckRole("admin") |
| DashboardController | /api/admin/dashboard/realtime-orders | @SaCheckLogin | @SaCheckRole("admin") |
| DashboardController | /api/admin/dashboard/orders | @SaCheckLogin | @SaCheckRole("admin") |
| AdminOrderController | 多个接口 | @SaCheckLogin + @SaCheckRole("admin") | ✅ 正确 |
| AdminUserController | 多个接口 | @SaCheckLogin + @SaCheckRole("admin") | ✅ 正确 |
| UserManagementController | 多个接口 | @SaCheckLogin | ❌ 缺少角色检查 |
| SchedulerController | /api/admin/scheduler/* | @SaCheckLogin | ❌ 缺少角色检查 |

#### 2.2.3 攻击场景示例

```bash
# 1. 商家登录获取Token
curl -X POST http://localhost:9090/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800000002","password":"123456"}'
# 返回: {"code":200,"data":{"token":"merchant_token"}}

# 2. 使用商家Token访问管理端接口
curl -X GET http://localhost:9090/api/admin/dashboard/overview \
  -H "Authorization: Bearer merchant_token"
# ❌ 当前: 返回200，可查看系统统计数据
# ✅ 应该: 返回403，无权限访问
```

#### 2.2.4 风险等级

| 风险维度 | 评级 | 说明 |
|----------|------|------|
| 机密性 | 高 | 敏感数据泄露风险 |
| 完整性 | 中 | 数据可能被非法修改 |
| 可用性 | 低 | 不影响系统可用性 |
| **综合风险** | **严重** | 需立即修复 |


### 3.2 当前实现方式

**认证方式**: JWT Token (Sa-Token框架)

**角色定义** (硬编码在User实体):
```java
public class User {
    private String role;  // 值: "admin", "merchant", "customer"
}
```

**权限校验方式**:
```java
// 方式1: 仅检查登录
@SaCheckLogin
public Result<?> someApi() { }

// 方式2: 检查特定角色
@SaCheckRole("admin")
public Result<?> adminApi() { }

// 方式3: 代码中手动检查
Long userId = StpUtil.getLoginIdAsLong();
User user = userMapper.selectOneById(userId);
if (!"admin".equals(user.getRole())) {
    throw new BusinessException(ErrorCode.USER_NO_PERMISSION);
}
```

### 3.3 权限模型对比

| 特性 | 当前实现 | 标准RBAC | 差距 |
|------|----------|----------|------|
| 角色定义 | 硬编码字符串 | 独立角色表 | ❌ 缺少 |
| 权限定义 | 无 | 独立权限表 | ❌ 缺少 |
| 角色-权限关联 | 无 | 关联表 | ❌ 缺少 |
| 用户-角色关联 | 单字段 | 关联表 | ⚠️ 简化 |
| 细粒度权限 | 无 | 支持 | ❌ 缺少 |
| 权限继承 | 无 | 支持 | ❌ 缺少 |
| 动态权限 | 无 | 支持 | ❌ 缺少 |

### 4.2 详细缺陷分析

#### 4.2.1 DEF-001: 管理端接口缺少角色校验

**问题代码**:
```java
// DashboardController.java
@SaCheckLogin  // 仅检查登录
@GetMapping("/overview")
public Result<Map<String, Object>> getOverview() { }

// UserManagementController.java
@SaCheckLogin  // 仅检查登录
@GetMapping("/list")
public Result<Map<String, Object>> getUserList() { }
```

**风险**:
- 普通用户可查看系统统计数据
- 商家可查看其他商家数据
- 可能导致敏感信息泄露

**修复代码**:
```java
@SaCheckRole("admin")  // 检查管理员角色
@GetMapping("/overview")
public Result<Map<String, Object>> getOverview() { }
```

#### 4.2.2 DEF-002: 无独立角色/权限表

**当前设计**:
```sql
-- user表
CREATE TABLE user (
    id BIGINT PRIMARY KEY,
    phone VARCHAR(20),
    password VARCHAR(100),
    role VARCHAR(20),  -- 单字段存储角色
    -- ...
);
```

**问题**:
- 角色定义分散在代码中
- 无法动态配置角色
- 无法支持复杂权限场景
- 难以扩展新角色

**标准RBAC设计**:
```sql
-- 角色表
CREATE TABLE role (
    id BIGINT PRIMARY KEY,
    code VARCHAR(50) UNIQUE,      -- 角色编码: admin, merchant, customer
    name VARCHAR(100),             -- 角色名称
    description VARCHAR(500),
    status TINYINT DEFAULT 1,
    create_time DATETIME,
    update_time DATETIME
);

-- 权限表
CREATE TABLE permission (
    id BIGINT PRIMARY KEY,
    code VARCHAR(100) UNIQUE,     -- 权限编码: user:create, order:view
    name VARCHAR(100),
    type VARCHAR(20),              -- menu, button, api
    resource VARCHAR(200),         -- 资源路径
    action VARCHAR(20),            -- create, read, update, delete
    status TINYINT DEFAULT 1
);

-- 角色-权限关联表
CREATE TABLE role_permission (
    role_id BIGINT,
    permission_id BIGINT,
    PRIMARY KEY (role_id, permission_id)
);

-- 用户-角色关联表
CREATE TABLE user_role (
    user_id BIGINT,
    role_id BIGINT,
    PRIMARY KEY (user_id, role_id)
);
```

#### 4.2.3 DEF-003: 密码哈希算法

**当前实现**:
```java
// BCrypt with default strength (10 rounds)
BCrypt.hashpw(password, BCrypt.gensalt());
```

**建议升级**:
```java
// 增加工作因子到12
BCrypt.hashpw(password, BCrypt.gensalt(12));

// 或考虑更安全的Argon2
Argon2 argon2 = Argon2Factory.create();
String hash = argon2.hash(10, 65536, 1, password);
```

#### 4.2.4 DEF-004: 无权限操作审计

**当前状态**: 无审计日志

**建议实现**:
```java
@Aspect
@Component
public class PermissionAuditAspect {
    
    @Autowired
    private AuditLogMapper auditLogMapper;
    
    @Around("@annotation(cn.dev33.satoken.annotation.SaCheckRole)")
    public Object auditRoleCheck(ProceedingJoinPoint point) throws Throwable {
        // 记录权限检查
        AuditLog log = new AuditLog();
        log.setUserId(StpUtil.getLoginIdAsLong());
        log.setAction("ROLE_CHECK");
        log.setResource(point.getSignature().toString());
        log.setTimestamp(LocalDateTime.now());
        
        try {
            Object result = point.proceed();
            log.setStatus("SUCCESS");
            return result;
        } catch (Exception e) {
            log.setStatus("FAILURE");
            log.setErrorMessage(e.getMessage());
            throw e;
        } finally {
            auditLogMapper.insert(log);
        }
    }
}
```

**权限服务实现**:

```java
@Service
public class PermissionService {
    
    @Autowired
    private UserRoleMapper userRoleMapper;
    
    @Autowired
    private RolePermissionMapper rolePermissionMapper;
    
    @Autowired
    private PermissionMapper permissionMapper;
    
    /**
     * 获取用户所有权限
     */
    public List<Permission> getUserPermissions(Long userId) {
        // 1. 获取用户角色
        List<Long> roleIds = userRoleMapper.selectListByQuery(
            QueryWrapper.create().where("user_id = ?", userId)
        ).stream().map(UserRole::getRoleId).collect(Collectors.toList());
        
        if (roleIds.isEmpty()) {
            return Collections.emptyList();
        }
        
        // 2. 获取角色权限
        List<Long> permissionIds = rolePermissionMapper.selectListByQuery(
            QueryWrapper.create().where("role_id IN ?", roleIds)
        ).stream().map(RolePermission::getPermissionId).distinct().collect(Collectors.toList());
        
        // 3. 获取权限详情
        return permissionMapper.selectListByIds(permissionIds);
    }
    
    /**
     * 检查用户是否有指定权限
     */
    public boolean hasPermission(Long userId, String permissionCode) {
        List<Permission> permissions = getUserPermissions(userId);
        return permissions.stream().anyMatch(p -> p.getCode().equals(permissionCode));
    }
}
```

**自定义权限注解**:

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface RequirePermission {
    String value();  // 权限编码
}

@Aspect
@Component
public class PermissionAspect {
    
    @Autowired
    private PermissionService permissionService;
    
    @Around("@annotation(requirePermission)")
    public Object checkPermission(ProceedingJoinPoint point, RequirePermission requirePermission) throws Throwable {
        Long userId = StpUtil.getLoginIdAsLong();
        String permissionCode = requirePermission.value();
        
        if (!permissionService.hasPermission(userId, permissionCode)) {
            throw new BusinessException(ErrorCode.USER_NO_PERMISSION, 
                "缺少权限: " + permissionCode);
        }
        
        return point.proceed();
    }
}
```

#### 5.2.3 REC-003: 增强密码安全 (P2)

**当前问题**:
- BCrypt默认工作因子为10
- 缺少密码复杂度校验
- 无密码历史记录

**改进方案**:

```java
@Component
public class PasswordEncoder {
    
    private static final int BCRYPT_STRENGTH = 12;  // 增加工作因子
    
    /**
     * 加密密码
     */
    public String encode(String rawPassword) {
        return BCrypt.hashpw(rawPassword, BCrypt.gensalt(BCRYPT_STRENGTH));
    }
    
    /**
     * 验证密码
     */
    public boolean matches(String rawPassword, String encodedPassword) {
        return BCrypt.checkpw(rawPassword, encodedPassword);
    }
    
    /**
     * 验证密码复杂度
     */
    public void validatePasswordStrength(String password) {
        if (password == null || password.length() < 8) {
            throw new BusinessException(ErrorCode.PARAM_ERROR, "密码长度至少8位");
        }
        
        // 检查包含大写字母
        if (!password.matches(".*[A-Z].*")) {
            throw new BusinessException(ErrorCode.PARAM_ERROR, "密码需包含大写字母");
        }
        
        // 检查包含小写字母
        if (!password.matches(".*[a-z].*")) {
            throw new BusinessException(ErrorCode.PARAM_ERROR, "密码需包含小写字母");
        }
        
        // 检查包含数字
        if (!password.matches(".*\\d.*")) {
            throw new BusinessException(ErrorCode.PARAM_ERROR, "密码需包含数字");
        }
        
        // 检查包含特殊字符
        if (!password.matches(".*[!@#$%^&*()].*")) {
            throw new BusinessException(ErrorCode.PARAM_ERROR, "密码需包含特殊字符");
        }
    }
}
```

#### 5.2.4 REC-004: 实现数据行级权限 (P1)

**场景**: 商家只能查看自己的订单，不能查看其他商家的订单

**实现方案**:

```java
@Aspect
@Component
public class DataPermissionAspect {
    
    @Autowired
    private UserMapper userMapper;
    
    /**
     * 拦截商家端查询方法，自动添加商家ID过滤
     */
    @Before("execution(* com.xingchen.backend.service.MerchantOrderService.*(..))")
    public void addMerchantFilter(JoinPoint point) {
        Long userId = StpUtil.getLoginIdAsLong();
        User user = userMapper.selectOneById(userId);
        
        if (user == null || user.getMerchantId() == null) {
            throw new BusinessException(ErrorCode.USER_NO_PERMISSION, "非商家用户");
        }
        
        // 将商家ID存入ThreadLocal，供后续查询使用
        DataPermissionContext.setMerchantId(user.getMerchantId());
    }
}

/**
 * 数据权限上下文
 */
public class DataPermissionContext {
    private static final ThreadLocal<Long> MERCHANT_ID = new ThreadLocal<>();
    
    public static void setMerchantId(Long merchantId) {
        MERCHANT_ID.set(merchantId);
    }
    
    public static Long getMerchantId() {
        return MERCHANT_ID.get();
    }
    
    public static void clear() {
        MERCHANT_ID.remove();
    }
}

/**
 * 在Mapper中使用
 */
@Mapper
public interface OrderMapper extends BaseMapper<Order> {
    
    default List<Order> selectByMerchant() {
        Long merchantId = DataPermissionContext.getMerchantId();
        if (merchantId == null) {
            throw new BusinessException(ErrorCode.USER_NO_PERMISSION);
        }
        return selectListByQuery(
            QueryWrapper.create()
                .where("merchant_id = ?", merchantId)
                .and("is_deleted = ?", 0)
        );
    }
}
```

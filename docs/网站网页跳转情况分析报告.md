## 1. 概述

### 1.1 系统架构

本系统采用 React + React Router 构建单页应用（SPA），包含三个主要用户端：

- **用户端 (User)**: 面向学生/普通用户的点餐界面
- **商家端 (Merchant)**: 面向商家的订单管理和店铺运营界面
- **管理端 (Admin)**: 面向平台管理员的后台管理界面

### 1.2 技术栈

| 技术项 | 版本/说明 |
|--------|-----------|
| 前端框架 | React 18 + TypeScript |
| 路由管理 | React Router v6 (createBrowserRouter) |
| 状态管理 | Zustand |
| UI组件库 | shadcn/ui + Tailwind CSS |
| 动画库 | Framer Motion |
| 数据获取 | TanStack Query (React Query) |

### 3.2 认证页面跳转

#### 3.2.1 登录页 (Login)

| 跳转目标 | 触发条件 | 跳转方式 | 参数说明 |
|----------|----------|----------|----------|
| / (首页) | 登录成功 | navigate('/') | 默认跳转 |
| /admin | 管理员登录成功 | navigate('/admin') | 根据角色跳转 |
| /merchant | 商家登录成功 | navigate('/merchant') | 根据角色跳转 |
| location.state.from | 有来源页面时 | navigate(from) | 登录后返回原页面 |
| /register | 点击"立即注册" | Link组件 | 无 |

**登录后跳转逻辑**:
```tsx
const handleSubmit = async (e) => {
  const response = await loginApi({ phone, password });
  login(response.user, response.token);
  
  // 根据角色跳转
  const from = location.state?.from?.pathname;
  if (from) {
    navigate(from, { replace: true });
  } else if (response.user.role === 'admin') {
    navigate('/admin');
  } else if (response.user.role === 'merchant') {
    navigate('/merchant');
  } else {
    navigate('/');
  }
};
```

## 4. 权限控制与路由守卫

### 4.1 路由守卫配置

```tsx
// RouteGuard 组件配置
interface RouteGuardProps {
  children: React.ReactNode;
  allowedRoles?: UserRole[];      // 允许的角色
  requireAuth?: boolean;          // 是否需要认证
  fallback?: React.ReactNode;     // 无权限时显示内容
}
```

### 4.2 权限检查流程

```
用户访问受保护页面
        ↓
[RouteGuard] 检查是否加载中
        ↓
检查 requireAuth 是否为 true
        ↓
    是 → 检查 isAuthenticated
        ↓
    未登录 → 跳转 /login (保存 from 路径)
        ↓
    已登录 → 检查 allowedRoles
        ↓
    有权限 → 显示页面内容
        ↓
    无权限 → 显示 fallback 或跳转默认页
```

### 4.3 各端权限要求

| 页面路径 | 需要登录 | 允许角色 | 无权限处理 |
|----------|----------|----------|------------|
| /, /stores, /store/:id | 否 | all | - |
| /cart | 否 | all | - |
| /checkout, /orders, /profile | 是 | user/merchant/admin | 跳转 /login |
| /payment/:orderNo | 是 | user/merchant/admin | 跳转 /login |
| /merchant/* | 是 | merchant | 显示权限提示页 |
| /admin/* | 是 | admin | 显示权限提示页 |

### 4.4 权限提示页面

**商家无权限提示**:
```tsx

  <p>您需要商家账号才能访问此页面</p>

</div>
```

**管理员无权限提示**:

## 5. 参数传递机制

### 5.1 URL 路径参数 (Params)

| 路由 | 参数名 | 类型 | 示例 |
|------|--------|------|------|
| /store/:id | id | string/number | /store/123 |
| /order/:id | id | string/number | /order/456 |
| /payment/:orderNo | orderNo | string | /payment/ORDER20240224001 |
| /address/edit/:id | id | string/number | /address/edit/789 |

**获取方式**:
```tsx
import { useParams } from 'react-router-dom';

function StoreDetail() {
  const { id } = useParams<{ id: string }>();
  // id = "123"
}
```

### 5.2 URL 查询参数 (Query String)

| 页面 | 参数名 | 用途 | 示例 |
|------|--------|------|------|
| /stores | categoryId | 分类筛选 | /stores?categoryId=1 |
| /stores | keyword | 搜索关键词 | /stores?keyword=奶茶 |
| /merchant/orders | status | 订单状态筛选 | /merchant/orders?status=2 |

**获取方式**:
```tsx
import { useSearchParams } from 'react-router-dom';

function Stores() {
  const [searchParams, setSearchParams] = useSearchParams();
  const categoryId = searchParams.get('categoryId');
  const keyword = searchParams.get('keyword');
}
```

### 5.3 状态传递 (State)

主要用于登录后的返回跳转：

```tsx
// 路由守卫中保存来源路径
<Navigate to="/login" state={{ from: location }} replace />

// 登录页面获取并跳转
const location = useLocation();
const from = location.state?.from?.pathname || '/';
navigate(from, { replace: true });
```

### 5.4 全局状态传递

使用 Zustand 存储跨页面状态：

```tsx
// 购物车状态
const useCartStore = create((set, get) => ({
  items: [],
  addItem: (item) => set((state) => ({ items: [...state.items, item] })),
```

## 6. 跳转延迟与性能分析

### 6.1 页面加载策略

| 页面类型 | 加载方式 | 首屏延迟 | 说明 |
|----------|----------|----------|------|
| Home | 立即加载 | < 100ms | 首页关键路径 |
| Login | 立即加载 | < 100ms | 认证关键路径 |
| 其他用户页 | 懒加载 (lazy) | 200-500ms | 代码分割 |
| 商家端页面 | 懒加载 | 300-600ms | 代码分割 |
| 管理端页面 | 懒加载 | 300-600ms | 代码分割 |

### 6.3 性能优化措施

1. **预加载关键页面**
   ```tsx

   useEffect(() => {

       import('@/pages/user/Stores');
   ```

   - 首页: StoreCardSkeleton
   - 商家页: StoreCardSkeleton
   - 订单页: OrderCardSkeleton
   - 仪表盘: DashboardSkeleton

3. **数据预取**
   ```tsx

     defaultOptions: {

       },
   ```

### 6.4 跳转延迟统计

| 操作类型 | 平均延迟 | 最大延迟 | 优化建议 |
|----------|----------|----------|----------|
| 首页→商家列表 | ~50ms | ~100ms | 已优化 |
| 商家列表→详情 | ~30ms | ~80ms | 已优化 |

## 7. 异常处理机制

### 7.1 错误边界 (Error Boundary)

```tsx
class ErrorBoundary extends React.Component {
  state = { hasError: false, error: null };
  
  static getDerivedStateFromError(error) {
    return { hasError: true, error };
  }
  
  componentDidCatch(error, errorInfo) {
    console.error('Error caught:', error, errorInfo);
  }
  
  render() {
    if (this.state.hasError) {
      return <ErrorFallback error={this.state.error} />;
    }
    return this.props.children;
  }
}
```

### 7.2 404 页面处理

```tsx
{
  path: '*',
  element: <Navigate to="/" replace />,
}
```

所有未匹配路由自动跳转至首页。

### 7.3 网络错误处理

```tsx
// API 请求错误统一处理
const handleApiError = (error) => {
  if (error.response?.status === 401) {
    // Token 过期，跳转登录
    navigate('/login');
  } else if (error.response?.status === 403) {
    // 权限不足
    toast.error('权限不足');
  } else if (error.response?.status >= 500) {
    // 服务器错误
    toast.error('服务器繁忙，请稍后重试');
  } else {
    toast.error(error.message || '操作失败');
  }
};
```

### 7.4 加载超时处理

```tsx
// Suspense 超时处理
<Suspense fallback={<PageSkeleton />}>
  <ErrorBoundary>
    <Outlet />
  </ErrorBoundary>
</Suspense>
```

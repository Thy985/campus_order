# 校园点餐管理系统数据库设计（降级版）

## 1. 数据库设计原则

### 1.1 设计原则
- **简单实用**：满足基本业务需求，避免过度设计
- **规范化**：遵循第三范式，减少数据冗余
- **性能优先**：合理设计索引，优化查询效率
- **可维护性**：表结构清晰，命名规范
- **安全可靠**：敏感数据加密存储

### 1.2 命名规范
- **表名**：小写字母 + 下划线，单数形式（如：user、product）
- **字段名**：小写字母 + 下划线
- **索引名**：idx_表名_字段名（如：idx_user_phone）
- **唯一索引**：uk_表名_字段名（如：uk_user_phone）

### 1.3 字段规范
- **主键**：BIGINT UNSIGNED，自增
- **时间字段**：DATETIME，默认值CURRENT_TIMESTAMP
- **金额字段**：DECIMAL(10,2)，单位：元
- **状态字段**：TINYINT，枚举值
- **逻辑删除**：is_deleted TINYINT，0-未删除，1-已删除
- **字符集**：utf8mb4（支持emoji）

### 2.2 核心表设计

#### 2.2.1 用户表 (user)
```sql

  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '用户ID',
  `username` VARCHAR(50) NOT NULL COMMENT '用户名',
  `nickname` VARCHAR(50) DEFAULT NULL COMMENT '昵称',
  `phone` VARCHAR(20) NOT NULL COMMENT '手机号',
  `password` VARCHAR(100) NOT NULL COMMENT '密码(BCrypt加密)',
  `avatar` VARCHAR(255) DEFAULT NULL COMMENT '头像URL',
  `gender` TINYINT DEFAULT 0 COMMENT '性别:0-未知,1-男,2-女',
  `user_type` TINYINT DEFAULT 1 COMMENT '用户类型:1-学生,2-教师,3-商家,4-管理员',
  `status` TINYINT DEFAULT 1 COMMENT '状态:0-禁用,1-正常',
  `last_login_time` DATETIME DEFAULT NULL COMMENT '最后登录时间',
  `last_login_ip` VARCHAR(50) DEFAULT NULL COMMENT '最后登录IP',
  `is_deleted` TINYINT DEFAULT 0 COMMENT '是否删除:0-否,1-是',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_phone` (`phone`),
  KEY `idx_user_type` (`user_type`),
  KEY `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

## 3. 索引设计策略

### 3.1 索引设计原则
- **主键索引**：每张表必须有主键
- **唯一索引**：唯一性字段（手机号、订单号等）
- **普通索引**：高频查询字段（user_id、merchant_id、status等）
- **联合索引**：多字段组合查询（遵循最左前缀原则）

### 3.2 核心索引列表
```sql
-- 用户表
idx_user_phone (phone)                    -- 手机号登录
idx_user_status (user_id, status)        -- 用户状态查询

-- 商家表
idx_merchant_category (category_id)       -- 分类查询
idx_merchant_status (status)             -- 状态查询

-- 商品表
idx_product_merchant (merchant_id)       -- 商家商品列表
idx_product_status (status)              -- 上下架查询

-- 订单表
uk_order_no (order_no)                   -- 订单号唯一
idx_order_user (user_id)                 -- 用户订单列表
idx_order_merchant (merchant_id)         -- 商家订单列表
idx_order_status (status)                -- 订单状态筛选
idx_order_time (create_time)             -- 时间范围查询

-- 支付表
uk_payment_no (payment_no)               -- 支付单号唯一
idx_payment_order (order_id)             -- 订单支付查询
```

## 5. 数据安全设计

### 5.1 敏感数据处理
```sql
-- 密码加密存储
UPDATE `user` SET password = BCrypt.hashpw('password', BCrypt.gensalt());

-- 手机号脱敏处理
-- 存储完整手机号，查询时脱敏
SELECT CONCAT(LEFT(phone, 3), '****', RIGHT(phone, 4)) AS phone FROM `user`;
```

### 5.2 权限控制
- **数据库用户**：最小权限原则
- **只读用户**：用于报表查询
- **读写用户**：用于应用操作
- **管理用户**：用于维护操作

### 5.3 SQL注入防护
- **参数化查询**：使用MyBatis参数绑定
- **输入验证**：前端和后端双重验证
- **转义处理**：特殊字符转义

## 6. 性能优化策略

### 6.1 查询优化
```sql
-- 1. 使用索引
SELECT * FROM `order` WHERE user_id = ? AND status = ?;

-- 2. 避免全表扫描
-- 错误示例: SELECT * FROM `order` WHERE create_time > '2024-01-01';
-- 正确示例: SELECT id, order_no FROM `order` WHERE create_time > '2024-01-01';

-- 3. 分页查询
SELECT * FROM `order` WHERE user_id = ? ORDER BY create_time DESC LIMIT 0, 20;

-- 4. 批量操作
INSERT INTO order_item VALUES (...), (...), (...);
```

### 6.2 索引优化
- **定期分析**：使用ANALYZE TABLE分析表统计信息
- **索引重建**：定期重建碎片化索引
- **覆盖索引**：使用索引覆盖查询字段
